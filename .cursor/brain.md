# Cursor Brain – Berntsen Struktur

## Project identity
Name: Berntsen Struktur  
Domain: berntsenstruktur.no  
Purpose: A consulting landing page that helps people structure ideas, startups and projects before they commit time or money.

Core message:
"Structure before risk."

The site should communicate clarity, honesty and competence.

---

## Tech stack

Frontend:
- HTML5
- CSS3
- minimal JavaScript

Hosting:
- GitHub Pages

No frameworks unless explicitly requested.

---

## File structure

index.html  
Main landing page structure.

styles.css  
All visual styling.

script.js  
Minimal interactions.

assets/  
Images and icons used by the website.

.cursor/  
Cursor configuration and project knowledge.

---

## Landing page architecture

The landing page must always follow this order:

1. Hero  
2. Services  
3. Process  
4. Who this is for  
5. Contact / Call-to-Action

Do not change this structure without explicit instruction.

---

## Design principles

The website should be:

- clean
- fast loading
- minimal
- readable
- trustworthy

Guidelines:

- Use semantic HTML.
- Avoid unnecessary animations.
- Prefer clarity over visual complexity.
- Keep CSS organized and readable.
- Avoid large frameworks or build systems.

---

## Editing rules

When modifying the repository:

1. Analyze the project structure first.
2. Maintain the existing layout unless asked to redesign.
3. Prefer improving existing code instead of rewriting everything.
4. Do not introduce dependencies without approval.
5. Keep the project compatible with GitHub Pages.

---

## Language rules

Explain changes in Norwegian.

Write all code in English.

---

## Behavior rules for Cursor

When working in this repository:

- Understand the project context before editing files.
- If unsure, ask for clarification instead of guessing.
- Make minimal and safe changes.
- Preserve working functionality.

Before making large changes:

1. Explain the plan.
2. Then implement.

---
